功能介紹
你需要的練習功能，全都有

從題庫匯入到進度追蹤，一個 HTML 檔案搞定所有事。

🎲
多種出題模式
支援全部隨機、按章節出題、章節隨機、重要題目、錯誤題目五種練習模式，針對弱點加強。
⭐
星星熟練系統
每答對一次累積一顆星（最多五顆），答錯清零。達到五顆星代表熟練，首頁即時統計熟練題數。
❤️
重要題目標記
答題途中隨時點愛心標記重要題目，之後可以專門針對重要題目練習，與錯誤標記分開管理。
❌
錯誤題自動記錄
答錯時自動標記，首頁顯示錯誤題數統計，可以單獨練習所有答錯過的題目，歷史累積保留。
📊
章節熟悉度進度條
每個章節顯示進度條、百分比、已答對題數，以及合格率（答對數÷已做過題數），一眼掌握學習狀況。
📝
個人筆記欄位
答完題看到解析後，可以輸入自己的筆記和重點，下次做同一題時自動顯示，備份時一起匯出。
📥
彈性題庫匯入
支援上傳 .json 檔案或直接貼上 JSON 文字，支援單選題、多選題、是非題三種題型，可含解析。
💾
備份與還原
一鍵匯出備份檔案（含題目、星星、愛心、筆記），換電腦或手機時匯入備份即可完整還原。
📱
PWA 手機 App
支援加入手機主畫面，像真正的 App 一樣使用，全螢幕顯示、離線可用，iOS 和 Android 都支援。
部署教學
5 分鐘部署到 GitHub Pages

不需要伺服器、不需要付費，只要有 GitHub 帳號就能免費架設。

💡
需要準備：一個免費的 GitHub 帳號，以及下載好的系統檔案（quiz-pwa.zip）。
1
建立 GitHub Repository
登入 GitHub，點右上角「＋」→「New repository」，名稱填 quiz-app，選 Public（公開），按「Create repository」建立。
2
上傳系統檔案
將 quiz-pwa.zip 解壓縮，取得以下 5 個檔案，全部上傳到 repo：
📄 index.html — 主程式（必要）
📄 manifest.json — App 設定（必要）
📄 sw.js — 離線支援（必要）
🖼 icon-192.png — App 小圖示（必要）
🖼 icon-512.png — App 大圖示（必要）
上傳方式：點「Add file」→「Upload files」，拖曳全部 5 個檔案，填寫 Commit 訊息，按「Commit changes」。
3
開啟 GitHub Pages
進入 repo 的「Settings」→ 左側「Pages」→ Source 選「Deploy from a branch」→ Branch 選「main」，資料夾選「/ (root)」→ 按「Save」。
⏳
等待約 1～2 分鐘，右側 Deployments 出現綠色 ✅ 即代表部署完成。
4
取得你的專屬網址
你的系統網址格式為：
https://你的GitHub帳號.github.io/quiz-app
用任何瀏覽器打開這個網址，就可以開始使用！
5
加入手機主畫面（變成 App）
用手機瀏覽器打開網址後：
📱 iPhone (Safari)：點下方分享按鈕 ⬆️ →「加入主畫面」→「新增」
🤖 Android (Chrome)：點右上角選單 ⋮ →「新增至主畫面」→「新增」
✅
加入後桌面會出現 📚 圖示，點開就像真正的 App，全螢幕顯示、離線也能使用！

題庫格式
JSON 題庫格式說明

使用標準 JSON 陣列格式，支援三種題型，可以分章節管理。

💡
匯入方式：點首頁右上角「＋」→ 上傳 .json 檔案 或 貼上 JSON 文字 → 按「匯入題庫」。重複匯入同一題（相同 id）會自動更新，不會重複。
題型type 值answer 格式需要 options
單選題 mc 單一字母，如 "A" ✅ 必填
多選題 multi 字母陣列，如 ["A","C"] ✅ 必填
是非題 tf "T" 或 "F" ❌ 不需要
questions.json
[ { // 單選題範例 "chapter": "第一章：基礎概念", "type": "mc", "question": "雲端運算中，IaaS 代表什麼？", "options": [ "Infrastructure as a Service", "Internet as a Service", "Integration as a Service", "Information as a Service" ], "answer": "A", "explanation": "IaaS（基礎設施即服務）是雲端服務模型之一。" }, { // 多選題範例 "chapter": "第一章：基礎概念", "type": "multi", "question": "以下哪些是雲端服務模型？（多選）", "options": ["SaaS", "IaaS", "BaaS", "PaaS"], "answer": ["A", "B", "D"], "explanation": "SaaS、IaaS、PaaS 是三大雲端服務模型。" }, { // 是非題範例 "chapter": "第二章：安全", "type": "tf", "question": "雲端服務可以完全消除資安風險。", "answer": "F", "explanation": "風險只能降低，無法完全消除。" } ]
欄位必填說明
chapter選填章節名稱，省略時歸類為「未分類」
type選填題型：mc / multi / tf，省略時預設為 mc
question必填題目文字內容
options選填選項陣列，單選和多選題必填，是非題不需要
answer必填正確答案（見上方格式說明）
explanation選填解析說明，答完題後顯示
id選填唯一識別碼，省略時自動產生。重複匯入相同 id 會更新而非新增
⚠️
注意：從 Word 或 PDF 複製內容時，可能含有全形引號（" "）或特殊空白，系統會自動修正這些字元。若匯入失敗，請確認 JSON 格式正確，可使用線上 JSON 驗證工具（如 jsonlint.com）檢查。
常見問題
FAQ

使用上有疑問？這裡可能有答案。

換了手機或電腦，進度會消失嗎？ ▾
進度存在瀏覽器的本機儲存（localStorage），換裝置時需要先在舊裝置點 💾 匯出備份，在新裝置匯入備份即可完整還原所有題目、星星、愛心標記和筆記。
在 Google Drive 打開，為什麼題目不見了？ ▾
Google Drive 預覽 HTML 時每次都是獨立的環境，無法保存進度。建議使用 GitHub Pages 部署，用固定網址開啟，進度才能正確保存在瀏覽器中。
匯入時出現「JSON 格式錯誤」怎麼辦？ ▾
常見原因：1) JSON 內容被截斷不完整；2) 有非標準引號（系統會自動修正大多數情況）；3) 行尾多了逗號。可以把 JSON 貼到 jsonlint.com 驗證格式，找出問題所在。
多選題怎麼算分？部分答對算對嗎？ ▾
多選題採全對才算對，必須選出全部正確答案且不能多選。部分正確或多選都算錯誤，星星會清零並標記為曾答錯。
答對後「曾答錯」標記會消失嗎？ ▾
不會自動消失，錯誤標記是歷史累積記錄，答對後不清除。如果想清除錯誤標記，可以點 💾 →「標記歸零」，這會同時清除星星、愛心和錯誤標記。
可以更新系統（新增功能）嗎？ ▾
可以。更新時只需要到 GitHub repo，上傳新版的 index.html 覆蓋舊的即可。題目進度存在瀏覽器 localStorage，更新程式不會影響進度，但建議更新前先備份一次。
合格率是怎麼計算的？ ▾
合格率 = 答對過的題數 ÷ 已做過的題數（答對過 + 答錯過）。未做過的題目不計入計算，所以一開始做題時合格率可能偏低，隨著練習次數增加會趨於穩定。
